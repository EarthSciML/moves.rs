# v0.2.0 Planning

This document summarises the v0.2.0 milestone.

## What v0.2.0 is

v0.1.0 shipped a complete set of calculator implementations but did not produce
real emission rows — the data-plane wiring connecting the Parquet default DB to
the live calculator context was not yet complete. v0.2.0's defining objective is:

> **`moves run` produces emission rows that pass the canonical diff gate.**

Everything else in v0.2.0 builds on or supports that goal.

## Milestone definition

v0.2.0 is ready to tag when:

1. `moves run --runspec SampleRunSpec.xml` produces a non-empty `MOVESOutput.parquet`.
2. All 34 non-scale fixtures pass the canonical diff gate in CI.
3. `scale-county`, `scale-project`, and `scale-rates` are in the active regression suite
 and pass.
4. The WASM demo is live at the GitHub Pages URL.
5. Real (not framework-overhead) performance benchmarks are documented.

## Priority order

1. **Onroad wiring** — blocks regression gate, performance characterization, and RunSpec coverage expansion.
2. **NONROAD wiring** — blocks regression gate.
3. **Regression gate** — blocks release.
4. **Scale wiring** — blocks release.
5. **WASM deployment, performance characterization, EPA MOVES annual update, RunSpec coverage expansion** — independent; run in parallel with regression gate and scale wiring.

## What v0.2 does NOT include

- GPU or distributed-execution targets (explicitly out of scope)
- GUI tooling
- Regulatory validation
- Uncertainty / Monte Carlo
- Any calculator that wasn't already shipped in v0.1 (scope creep risk — open a work item and target v0.3 if a missing calculator surfaces from community feedback)
