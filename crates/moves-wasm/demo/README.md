# moves.rs WebAssembly Demo

A minimal browser-hosted demo that runs the MOVES Rust port entirely in the
browser — no server-side computation required after the initial page load.
This is a credible existence proof that the onroad and NONROAD simulation
engines work in a WASM context, not a polished UI.

## Quick start

### 1. Install wasm-pack

```bash
cargo install wasm-pack
```

### 2. Build the WASM package

From the **repo root**:

```bash
wasm-pack build --target web crates/moves-wasm
```

This places the generated JS glue and WASM binary in
`crates/moves-wasm/pkg/`.

### 3. Serve the demo

Browser security policy requires the files to be served over HTTP (not
loaded directly as `file:///`). Any static server works:

```bash
# Python 3 (no install needed)
python3 -m http.server 8080 --directory crates/moves-wasm

# Node.js (npx, no install needed)
npx serve crates/moves-wasm
```

Then open `http://localhost:8080/demo/` in your browser.

### 4. Run a simulation

**Onroad:**
1. Click "Choose file" next to *RunSpec XML*.
2. Select `characterization/fixtures/sample-runspec.xml` from the repo.
3. Leave *Max parallel chunks* at 0 (auto/sequential).
4. Click **Run onroad**.
5. After a few seconds the output files appear as download links.

**NONROAD:**
1. Click "Choose file" next to *Population file (.POP)*.
2. Select a NONROAD `.POP` population file.
3. Set *Episode year* and *Region level* as desired.
4. Optionally enter FIPS codes in *Selected counties*.
5. Click **Run NONROAD**.
6. Run counters appear when complete.

## Multi-threaded build

The default build runs sequentially. For the multi-threaded variant
(requires nightly Rust; page must be served with COEP/COOP headers):

```bash
RUSTFLAGS="-C target-feature=+atomics,+bulk-memory,+mutable-globals" \
 cargo +nightly build \
 -Z build-std=std,panic_abort \
 --target wasm32-unknown-unknown \
 --package moves-wasm \
 --features wasm-threads \
 --release

wasm-bindgen \
 --target web \
 --out-dir crates/moves-wasm/pkg/ \
 target/wasm32-unknown-unknown/release/moves_wasm.wasm
```

See [`docs/wasm-threading.md`](../../../docs/wasm-threading.md) for
full deployment instructions, including the required HTTP headers.

## File layout

```
crates/moves-wasm/
├── demo/
│ ├── index.html ← demo page (this directory)
│ ├── worker.js ← Web Worker that loads and runs the WASM module
│ └── README.md ← this file
├── pkg/ ← generated by wasm-pack (not committed)
│ ├── moves_wasm.js
│ ├── moves_wasm_bg.wasm
│ └── …
└── src/
 └── lib.rs ← WASM entry points (run_simulation, run_nonroad_simulation)
```

## Browser compatibility

| Browser | Onroad (single-thread) | NONROAD (single-thread) | Multi-thread |
|---------|----------------------|------------------------|--------------|
| Chrome 90+ | ✓ | ✓ | ✓ (with COEP/COOP headers) |
| Firefox 89+ | ✓ | ✓ | ✓ (with COEP/COOP headers) |
| Safari 15.2+ | ✓ | ✓ | ✓ (with COEP/COOP headers) |

ES modules in Web Workers (`type: "module"`) are required. Chromium-based
browsers support this broadly; Firefox 114+ and Safari 15.2+ added support.
For older browsers, use a bundler (Webpack, Rollup, Vite) to produce a
classic worker bundle — see [`docs/wasm-embedding.md`](../../../docs/wasm-embedding.md).
